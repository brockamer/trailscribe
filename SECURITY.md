# Security Policy

If you believe you have found a security vulnerability in Trailscribe, please report it privately.  We take security issues seriously and request that you do not publicly disclose a problem until it has been addressed.

## Supported versions

We currently maintain the latest major version of Trailscribe.  Security fixes are only provided for the main branch.

## Reporting a vulnerability

To report a vulnerability, please send an email to **[your‑email@example.com]** with details and steps to reproduce.  Include any relevant logs, stack traces or proof of concept code.  We will respond within a reasonable time frame and work with you to triage and address the issue.

In your report, please include:

* A description of the vulnerability and its potential impact
* Steps to reproduce or a proof of concept
* Any known mitigations or workarounds

Thank you for helping keep the project secure!
